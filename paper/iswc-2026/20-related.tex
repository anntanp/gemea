\section{Related Work}
\label{sec:related}

\textbf{CH knowledge graphs and LOD initiatives.}
Europeana LOD~\cite{haslhofer2013europeanalod} is the largest CH Linked Open Data initiative, exposing EDM-compliant metadata for tens of millions of objects aggregated from across Europe; its SPARQL endpoint has been discontinued and data is distributed primarily as bulk download, with no interactive graph browser.
The GND is accessible as a bulk download\footnote{GND Open Data, \url{https://data.dnb.de/opendata/}}, via the lobid REST API\footnote{hosted by Hochschulbibliothekszentrum des Landes Nordrhein-Westfalen (hbz), \url{https://lobid.org/gnd/api}} and via a QLever-hosted SPARQL endpoint,\footnote{\url{https://sparql.dnb.de/dnbgnd}} but none provides a Linked Data browser.

\textbf{Domain-specific CH knowledge graphs.}
ArCo~\cite{carriero2019arco} is an Italian CH KG modeled with CIDOC-CRM and interlinked with DBpedia and the Getty Vocabularies, covering around 820K cultural objects; it provides a SPARQL endpoint but no graph browser.
PHAROS\footnote{\url{https://artresearch.net/sparql}} is an international consortium of photo archives exposing millions of art-historical photographs via a public SPARQL endpoint and discovery interface; coverage is limited to visual art photography.

\textbf{ArtKB~\cite{blanco2026artkb} (ESWC 2026).}
ArtKB is a multimodal knowledge base for art objects built using the CACAO ontology (a CIDOC-CRM extension), combining a GraphDB RDF store with a Qdrant vector database, MinIO object storage, and an API gateway.
It supports Text2SPARQL, neural image retrieval, AI metadata suggestion, and Text2RDF.
ArtKB covers 307K curated art objects from Wikidata, exposes an API gateway without a public SPARQL endpoint, and does not document self-hosting or include a Linked Data browser.

\textbf{Ontology alignment.}
MOCHO~\cite{tan2026navigating} maps \texttt{edm:ProvidedCHO}\footnote{All CH objects in EDM are instances of this class.} to FRBR~\cite{ifla1998frbr} levels via GND Werk links, aligning DDB-EDM to domain- and modality-specific ontologies. GEMEA applies MOCHO to the full DDB corpus.

\textbf{SPARQL endpoints and KG browsers.}
Public SPARQL endpoints such as the Wikidata Query Service are domain-general; generic LOD browsers support triple dereferencing but not faceted search or collection navigation.
SHMARQL~\cite{shmarql} provides a Linked Data browser and SPARQL proxy deployable via Docker over any existing endpoint.
The DDB's own interface supports keyword search but provides no SPARQL access, graph navigation, or open data export.

Only GEMEA combines national-scale cross-domain coverage with a public SPARQL endpoint, an interactive Linked Data browser, and self-hostable deployment (Table~\ref{tab:related}).
The MCP/MCPO agentic access layer has no published precedent in cultural heritage KG work.
