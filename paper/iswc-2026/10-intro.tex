\section{Introduction}
\label{sec:introduction}

The German Digital Library (Deutsche Digitale Bibliothek, DDB)\todo{cite here!} aggregates over 65 million objects from archives, libraries, museums, media libraries, and monument preservation institutions across Germany, all described using the Europeana Data Model (EDM)\todo{cite here!}.
Despite this semantic foundation, the corpus has no public SPARQL endpoint, no Linked Data browser, and no machine-readable interface suitable for KG-based research.
Access is limited to the DDB's own search interface, which supports neither graph traversal nor structured querying.

A shared metadata schema is rarely encountered in its idealized form at scale.
Institutions encode data against different versions of the same schema, apply locally-specific interpretations of its fields, and carry cataloging traditions shaped by sector — library, archive, museum — into every record.
These local conventions are internally consistent; their collective complexity becomes visible only at transformation scale.

GeMeA (German Memory Atlas) is a publicly accessible knowledge graph over 26 million DDB objects.
The name reflects both the resource's scope and its function: the DDB aggregates the holdings of German memory institutions --- archives, libraries, museums, and media libraries --- and GeMeA provides the structured, graph-navigable index over that record that has so far been absent.
Three contributions are described in this paper:

\begin{enumerate}
    \item \textbf{Resource.} A publicly queryable KG of 26M German cultural heritage objects, served via a QLever SPARQL endpoint~\cite{bast2017qlever} and browsable through a SHMARQL Linked Data interface --- the first open, SPARQL-accessible KG over the DDB corpus.

    \item \textbf{Ontology alignment PoC.} A \texttt{mocho}-based alignment layer mapping DDB EDM metadata to RDA/FRBR-compatible representations, demonstrated on a representative subset, with scope boundaries documented rather than suppressed.

    \item \textbf{Agentic access layer.} An MCP/MCPO interface on the QLever endpoint, enabling AI agents to query GeMeA programmatically via any MCP-compatible client.
\end{enumerate}

GeMeA is released as a research testbed for LLM + KG research at cultural heritage scale, including KG-grounded retrieval, entity embedding, explainability, and agentic KG access.
Ontology evaluation on the \texttt{mocho}-aligned subgraph and provenance modeling of LLM-assisted KG enrichment are also supported.
Data is licensed CC~BY~4.0; the SPARQL endpoint, Linked Data browser, and source code are available at \todo{[persistent URI]}.
