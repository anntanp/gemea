# ISWC 2026 Resource Track — Review Checklist

Use this to self-assess the submission before each deadline. Based on the official review criteria.

---

## Mandatory Requirements

- [ ] Resource published at a **persistent URI** (PURL, DOI, w3id)
- [ ] **Canonical citation** associated with the resource
- [ ] **License** clearly specified (CC or OSI-approved)
- [ ] Resource accessible to reviewers (public or anonymous access)
- [ ] Paper uses **ISWC Resource Availability Statement** wording
- [ ] **Declaration of use of Generative AI** section included (before references)
- [ ] Abstract submitted by **2 May 2026**
- [ ] Full paper (8–15 pp) submitted by **7 May 2026**
- [ ] Formatted in **Springer LNCS** style

---

## Impact

- [ ] Does the resource break new ground?
- [ ] Does it fill an important gap (articulated clearly)?
- [ ] How does it advance the state of the art?
- [ ] Is it compared to existing resources of similar scope?
- [ ] Is it relevant to the semantic web community?
- [ ] What is its broader societal interest?
- [ ] How will it support adoption of semantic web technologies?

---

## Reusability & Reproducibility

- [ ] Evidence of use by a community beyond the authors (or strong potential if new)
- [ ] Is the resource easy to use? High-quality documentation?
- [ ] Tutorials available?
- [ ] Are evaluation results and metrics reproducible?
- [ ] Is it general enough for scenarios beyond the original design?
- [ ] Is there potential for extensibility?
- [ ] Clear explanation of how others should use it
- [ ] Clearly states what the resource **cannot** do and why

---

## Design & Technical Quality

- [ ] Follows resource-specific best practices
- [ ] Reuses or extends suitable high-quality existing resources
- [ ] Appropriate for solving the task at hand
- [ ] Human- and machine-readable description (FAIR)
- [ ] Schema diagram included
- [ ] Dataset described with VoID / DCAT / Dublin Core

---

## Availability

- [ ] Publicly available (SPARQL endpoint, download, open repo)
- [ ] Publicly findable (DataHub, Zenodo, or equivalent registry)
- [ ] Registered in generic repositories (Zenodo, GitHub)
- [ ] Sustainability plan specified (medium and long-term maintenance)
- [ ] Adopts open standards (RDF, SPARQL, EDM, RDA)
- [ ] Self-hosting option documented

---

## Optional but Positive

- [ ] ORKG comparison created and linked in submission
- [ ] Croissant metadata embedded in PDF
- [ ] Metadata embedded in PDF (machine-readable link to resource)
- [ ] VoID dataset descriptor published at resource URI
